Installation

Pre-built Binaries

Download the latest release from GitHub:

  • Windows: Openza-Setup-{version}.exe (installer) or Openza-Portable.exe
  • Linux: Openza_{version}_amd64.deb or Openza-{version}.AppImage

Build from Source

Requirements: Node.js 18+ and npm

# Clone the repository
git clone https://github.com/openza/openza-desktop.git
cd openza-desktop

# Install dependencies
npm install

# Start development server
npm run dev:electron

Quick Start

  1. 1

    Launch Openza Desktop

    Open the application after installation.

  2. 2

    Connect Your Task Provider

    Click "Sign in with Todoist" or "Sign in with Microsoft" on the login page.

  3. 3

    Authorize Access

    Complete the OAuth flow in your browser and return to the app.

  4. 4

    Start Managing Tasks

    View your tasks in the dashboard, organized by provider, project, and due date.

Configuration

For development or custom deployments, configure environment variables in .env.local:

# Todoist Configuration
VITE_TODOIST_CLIENT_ID=your_todoist_client_id
VITE_TODOIST_CLIENT_SECRET=your_todoist_client_secret
VITE_REDIRECT_URI=http://localhost:5173/auth/callback

# Microsoft To-Do Configuration
VITE_MSTODO_CLIENT_ID=your_microsoft_app_client_id
VITE_MSTODO_CLIENT_SECRET=your_microsoft_app_client_secret
VITE_MSTODO_REDIRECT_URI=http://localhost:3000/auth/mstodo/callback
VITE_MSTODO_TENANT_ID=common

# OAuth Protocol Configuration
VITE_OAUTH_PROTOCOL_SCHEME=openza
VITE_OAUTH_TIMEOUT_MINUTES=3

Security Warning: Never commit .env.local to version control. It contains sensitive credentials.

Todoist Integration Setup

  1. 1. Create Todoist App

    Go to Todoist App Console

  2. 2. Configure OAuth

    Set redirect URI:

    • Development: http://localhost:5173/auth/callback
    • Production: openza://auth/callback

  3. 3. Copy Credentials

    Add Client ID and Secret to your .env.local file

Microsoft To-Do Integration Setup

  1. 1. Register Azure AD App

    Go to Azure Portal - App Registrations

  2. 2. Configure Authentication

    • Add redirect URI: http://localhost:3000/auth/mstodo/callback
    • Enable "Accounts in any organizational directory and personal Microsoft accounts"

  3. 3. Create Client Secret

    In "Certificates & secrets", create a new client secret

  4. 4. Add API Permissions

    Add Microsoft Graph permissions: Tasks.ReadWrite, User.Read

Local Database

Openza uses SQLite with better-sqlite3 for offline task management. The database includes:

  • Full-text search (FTS5) for fast task searching
  • Automatic migrations for schema updates
  • Encrypted storage in Electron's app data directory
  • Offline-first task creation and editing

The database is automatically initialized on first run. No manual setup required.

Architecture

Openza follows a multi-layered architecture:

Key Components

  • TaskSourceContext: Manages multi-provider data sources
  • useUnifiedTasks: Aggregates tasks from all providers
  • Auth Manager: Centralized authentication with secure token storage
  • IPC Security: All sensitive operations in main process

Security

Security is a top priority. See our Security Policy for details.

Security Features

  • No hardcoded secrets
  • Encrypted token storage
  • IPC security (main process only for sensitive ops)
  • CSP and context isolation
  • OAuth 2.0 with PKCE
